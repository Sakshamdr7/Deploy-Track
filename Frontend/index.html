Project Summary

About Deploy-Track

A concise explanation of the project, its purpose, and the system flow for the end-term demo.

Purpose

Deploy-Track is built to monitor CI/CD activity, deployment records, and environment updates in one dashboard for a full-stack DevOps-focused minor project.

Core Workflow

Deployment events are created from the dashboard or CI pipeline, sent to the backend API, stored in persistent local storage, and rendered back into the frontend as activity and stats.

Tech Stack

Frontend uses HTML, CSS, and JavaScript. Backend uses Node.js and Express. Current persistence is JSON-based local storage, with CI handled through GitHub Actions.

End-Term Goal

Present a polished local website that looks like a real CI/CD monitoring product, with structured frontend navigation, backend API support, deployment history, and system status visibility.
